Phase 1: Setting Up Your Computer
Before you write a single line of code, you need to set up your development environment. This is a one-time process. Follow every step in order.


Part A: Install Core Programs


Step 1 — Install Node.js (Required for Everything)
Node.js lets your computer run JavaScript outside a browser. It is the foundation of the entire tech stack.


Step 1:  Go to nodejs.org and download Node.js
Open your web browser and go to:  https://nodejs.org
Click the big green button that says 'LTS' (Long Term Support)
This downloads an installer file to your Downloads folder
Open the downloaded file and click through the installer (click Next on everything)
When it finishes, close the installer
	

Now verify it installed correctly. Open your Terminal (Mac) or Command Prompt (Windows):
📌  NOTE: Opening Terminal on Mac
Press Command + Space to open Spotlight Search
Type 'Terminal' and press Enter
A black or white window opens — this is your Terminal
	

📌  NOTE: Opening Command Prompt on Windows
Press the Windows key
Type 'cmd' and press Enter
A black window opens — this is your Command Prompt
	

Type this command and press Enter:
node --version
	You should see something like:  v20.11.0   (any number is fine)
Then type this and press Enter:
npm --version
	You should see something like:  10.2.4   (any number is fine)


⚠️  WARNING: If you see 'command not found'
Close your Terminal/Command Prompt completely, reopen it, and try again.
If it still says 'not found', your installation did not work. Go back to nodejs.org and reinstall.
	

Step 2 — Install Git (Code Version Control)
Git saves snapshots of your code so you can go back if something breaks. It also connects to GitHub where your code lives in the cloud.


Step 2:  Install Git on your computer
MAC: Open Terminal and type:  xcode-select --install   then press Enter
     A popup will appear — click 'Install' and wait (takes 5-10 minutes)
WINDOWS: Go to https://git-scm.com/download/win
         Download the installer, run it, click Next through everything
         On the screen 'Choosing the default editor', select 'Use Visual Studio Code'
	

Verify Git installed:
git --version
	You should see something like:  git version 2.43.0


Step 3 — Install Cursor (Your AI Code Editor)
Cursor is like Microsoft Word but for code. It has Claude AI built directly into it, so you can highlight any code and ask questions about it, or tell it to fix bugs.


Step 3:  Download and install Cursor
Go to:  https://cursor.sh
Click 'Download for Mac' or 'Download for Windows'
MAC: Open the .dmg file, drag Cursor to your Applications folder
WINDOWS: Run the .exe installer, click through everything
Open Cursor — it will ask you to sign in or create an account
Create a free account with your email
When it asks about AI provider, select 'Claude' if available, or use the default
It will ask about your monthly plan — start with Free, upgrade to Pro ($20) when you are ready to build seriously
	

Step 4 — Install Expo CLI (The Mobile App Builder)
Expo is the tool that turns your code into a real iPhone and Android app. You install it using npm (which came with Node.js).


In your Terminal, type this exactly and press Enter:
npm install -g expo-cli
npm install -g eas-cli
	Wait for it to finish (1–2 minutes). You will see a lot of text scroll by — that is normal.


Verify Expo installed:
expo --version
	You should see a version number like:  6.3.10


Step 5 — Install the Expo Go App on Your Phone
Expo Go lets you see your app on your real phone instantly while you are building it — no App Store submission needed during development.
Step 5:  Get Expo Go on your phone
On your iPhone: Open the App Store, search 'Expo Go', install it
On Android: Open the Play Store, search 'Expo Go', install it
Open Expo Go — create a free account with the same email you used for Cursor
Leave the app open — you will use it constantly while building

Step 13 — Create Your Environment Variables File
Environment variables store your secret API keys so they are NOT hardcoded in your code (which would be a security risk).


In Cursor, look at the left sidebar (your file list). Right-click in the empty space and click 'New File'. Name it:  .env


In the .env file, paste this and fill in your actual keys:
# Supabase
EXPO_PUBLIC_SUPABASE_URL=https://YOURPROJECTID.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key_here


# Stripe
EXPO_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_your_key_here
STRIPE_SECRET_KEY=sk_test_your_key_here


# Plaid
PLAID_CLIENT_ID=your_plaid_client_id
PLAID_SECRET=your_plaid_sandbox_secret
PLAID_ENV=sandbox


# App
EXPO_PUBLIC_API_URL=http://localhost:3001
	

⚠️  WARNING: CRITICAL: Protect Your .env File
Right-click in the file list, create a new file named '.gitignore'
In that file, type exactly:  .env
This tells Git to NEVER upload your .env file to GitHub
If you skip this, your secret keys will be visible to the entire internet

Part C: Set Up Your Database Schema
Now you need to create all the database tables in Supabase. Open Claude in a new conversation and paste this:


PASTE THIS INTO CLAUDE — Database Schema Prompt
	I am building WealthWise Kids — a financial literacy app for children.
My database is PostgreSQL managed by Supabase.


Write all the SQL CREATE TABLE statements for the following tables.
For each table, include: column names, data types, constraints, primary keys,
foreign keys, and relevant indexes. Also include the Row Level Security (RLS)
policies for each table — this is critical for Supabase security.


Tables needed:


1. profiles
   - Extends Supabase auth.users
   - Fields: id (UUID, FK to auth.users), full_name, avatar_url,
     account_type ('parent' or 'child'), created_at
   - RLS: users can only read/write their own profile


2. child_profiles
   - Fields: id (UUID), parent_id (FK to profiles), name, birthdate,
     avatar_choice (integer 1-10), created_at
   - RLS: parent can CRUD their own children; children can read their own record


3. parental_consents
   - Fields: id, parent_id (FK), child_id (FK), consent_given (boolean),
     consent_timestamp, ip_address (text), created_at
   - RLS: only the parent can read their consents


4. subscriptions
   - Fields: id, user_id (FK to profiles), plan_type ('free','premium','family'),
     stripe_customer_id, stripe_subscription_id, status ('active','canceled','past_due'),
     current_period_end (timestamptz), created_at, updated_at
   - RLS: users can read their own subscription


5. schools
   - Fields: id (integer), title, description, order_number, icon_name,
     is_premium (boolean), min_age, created_at
   - RLS: public read access (everyone can see what schools exist)


6. lessons
   - Fields: id (UUID), school_id (FK), title, content (text — the lesson body),
     lesson_number (integer), lesson_type ('content','did_you_know','activity'),
     fun_fact (text nullable), created_at
   - RLS: public read access


7. quiz_questions
   - Fields: id (UUID), school_id (FK), lesson_id (FK nullable),
     question_text, question_type ('multiple_choice','true_false','fill_blank'),
     options (JSONB — array of option strings), correct_answer (text),
     explanation (text), difficulty ('easy','medium','hard'), order_number
   - RLS: public read access


8. student_progress
   - Fields: id, child_id (FK), school_id (FK), lesson_id (FK),
     completed (boolean), completed_at (timestamptz), created_at
   - Unique constraint on (child_id, lesson_id)
   - RLS: child can read their own; parent can read their children's


9. quiz_attempts
   - Fields: id, child_id (FK), school_id (FK), score (integer 0-100),
     passed (boolean), answers (JSONB), attempted_at, time_taken_seconds
   - RLS: child can read their own; parent can read their children's


10. badges
    - Fields: id, name, description, icon_name, rarity ('common','rare','epic'),
      trigger_type (text), trigger_value (text), created_at
    - RLS: public read


11. student_badges
    - Fields: id, child_id (FK), badge_id (FK), earned_at
    - Unique on (child_id, badge_id)
    - RLS: child can read their own; parent can read children's


12. wealth_coins
    - Fields: id, child_id (FK), balance (integer default 0),
      last_updated
    - One row per child
    - RLS: child reads own; parent reads children's


13. coin_transactions
    - Fields: id, child_id (FK), amount (integer positive or negative),
      reason (text), created_at
    - RLS: child reads own


14. streaks
    - Fields: id, child_id (FK), current_streak (integer), longest_streak,
      last_activity_date (date), freeze_available (boolean default true),
      last_freeze_used (date nullable)
    - One row per child
    - RLS: child reads own; parent reads children's


15. budget_entries
    - Fields: id, child_id (FK), entry_type ('income','expense'),
      amount (numeric), category (text), description (text), entry_date (date),
      is_from_plaid (boolean), plaid_transaction_id (text nullable), created_at
    - RLS: child reads/writes own; parent reads children's


16. plaid_connections
    - Fields: id, child_id (FK), access_token_encrypted (text),
      item_id (text), institution_name (text), last_synced (timestamptz),
      is_active (boolean), created_at
    - RLS: only the child's parent can read/write (extra security for financial data)


17. saving_goals
    - Fields: id, child_id (FK), title, target_amount (numeric),
      current_amount (numeric default 0), icon_emoji (text),
      target_date (date nullable), completed (boolean), created_at
    - RLS: child reads/writes own; parent reads


After the CREATE TABLE statements, write the SQL to insert seed data for:
- The 7 lessons for School 1 (Money Fundamentals)
- 5 quiz questions for School 1
- The 5 badge types for MVP
- School 1 and School 2 records in the schools table


Format everything clearly with comments separating each table.

	You should see:  {"status":"ok","timestamp":...}

When You Get Stuck — Escalation Ladder

26. Ask Claude one more time but say: 'The previous suggestions did not work. Here is the full error, full file, and everything I have tried. Please approach this differently.'
27. Search Stack Overflow for your exact error message — there is almost always an answer
28. Search the Expo Discord server (discord.gg/expo) — very active community
29. Search the Supabase GitHub issues if it is a database problem
